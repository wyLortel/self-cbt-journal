#### Issue RECORD-FORM-COMMON-001: 기록 작성 페이지 레이아웃
**복잡도**: 🟡 Mid | **소요시간**: 2시간

```markdown
# Issue: 기록 작성 페이지 기본 레이아웃 및 헤더

## 목적
9단계 단계적 입력을 위한 페이지 구조와 네비게이션 구현

## 설계 참고
- 최대 너비: max-w-2xl
- 배경: bg-white (단계별 하나씩 보여줌)
- 반응형: 모바일에서는 전체 화면 사용

## 완료 조건 (Acceptance Criteria)

### AC 1: 헤더 구성
- Given 기록 작성 페이지 로드
- When 페이지 상단 확인
- Then 다음이 보임:
  - 좌측: ← 뒤로가기 버튼 (onclick으로 이전 페이지 또는 모달 닫기)
  - 중앙: "현재 단계 / 9" (text-lg, font-bold, text-indigo-600)
  - 우측: 저장 상태 배지 ("임시 저장됨", "저장 실패" 등)

### AC 2: 진행도 바
- Given 헤더 아래
- When 진행도 확인
- Then 다음이 보임:
  - 가로 프로그레스 바 (h-2, bg-gray-200)
  - 채워진 부분 (bg-indigo-600, 너비 = 현재_단계/9 * 100%)
  - 예: 3단계 = 33% 채워짐

### AC 3: 콘텐츠 영역
- Given 진행도 바 아래
- When 화면 확인
- Then 다음이 보임:
  - px-6 py-8 (데스크톱), px-4 py-6 (모바일)
  - 흰 배경, rounded-none (전체 페이지)
  - 최대 너비 제한 (max-w-2xl, mx-auto)

### AC 4: 반응형
- Given 모바일 (375px)
- When 페이지 로드
- Then 패딩 줄어들고, 텍스트 크기 조정

### AC 5: 뒤로가기 버튼 동작
- Given 뒤로가기 버튼 클릭
- When 클릭 실행
- Then 이전 페이지로 이동 또는 모달 닫기
  + 작성 중인 내용은 Zustand에 임시 저장됨 (다시 진입하면 복구)
```

## 목적
각 단계의 제목과 설명을 일관되게 표시

## 완료 조건 (Acceptance Criteria)

### AC 1: 제목 렌더링

#### Issue RECORD-FORM-COMMON-003: 단계 네비게이션 버튼 (이전/다음)
**복잡도**: 🟡 Mid | **소요시간**: 1.5시간

```markdown
# Issue: 단계 네비게이션 버튼 (이전/다음)

## 목적
단계를 이동할 수 있는 버튼 및 검증 로직

## 완료 조건 (Acceptance Criteria)

### AC 1: 버튼 렌더링
- Given 각 단계의 하단
- When 버튼 확인
- Then 다음이 보임:
  - 좌측: [이전] 버튼 (Secondary Button, h-12)
    - 1단계에서는 비활성화 (또는 숨김)
  - 우측: [다음] 버튼 (Primary Button, h-12)

### AC 2: 다음 버튼 활성화 조건
- Given 현재 단계의 필수 필드가 모두 입력됨
- When 다음 버튼 확인
- Then 버튼이 활성화 (cursor-pointer, bg-indigo-600)

### AC 3: 다음 버튼 비활성화 조건
- Given 필수 필드가 비어있거나 유효하지 않음
- When 버튼 확인
- Then 버튼이 비활성화 (cursor-not-allowed, bg-gray-200, text-gray-400)

### AC 4: 이전 버튼 동작
- Given 사용자가 이전 버튼 클릭 (2단계 이상)
- When 클릭 실행
- Then 이전 단계로 이동, 입력 상태 유지 (Zustand)

### AC 5: 다음 버튼 동작
- Given 사용자가 다음 버튼 클릭
- When 클릭 실행
- Then 다음 단계로 이동, 입력 상태 저장

### AC 6: 마지막 단계 (9단계)
- Given 9단계에서 다음 버튼 확인
- When 버튼 텍스트 확인
- Then "저장" 또는 "완료"로 변경
  + 클릭 시 API로 저장 후 기록 목록으로 리다이렉트
```

#### Issue RECORD-003: [2단계] 감정 선택 - 감정 카드
**복잡도**: 🟡 Mid | **소요시간**: 3시간

```markdown
# Issue: 2단계 - 감정 선택 UI (7개 감정 카드)

## 목적
7가지 감정을 시각적으로 선택할 수 있는 인터페이스

## 설계
7개 감정: 불안, 자책, 수치심, 후회, 분노, 좌절, 외로움
- 각각 이모지 아이콘 포함
- 카드 형태 (w-[100px], h-[100px] 정도)
- 선택/미선택 상태 명확

## 완료 조건 (Acceptance Criteria)

### AC 1: 카드 렌더링
- Given 2단계 로드
- When 감정 선택 섹션 확인
- Then 다음이 보임:
  - 7개 감정 카드가 그리드로 배치 (grid-cols-4 md:grid-cols-7)
  - 각 카드: rounded-2xl, border, 중앙 정렬

### AC 2: 감정 카드 스타일
- Given 각 감정 카드
- When 카드 확인
- Then 다음이 보임:
  - 상단: 큰 이모지 (text-4xl)
  - 하단: 감정명 (text-sm, font-semibold)
  - 배경: bg-white, border-gray-200

### AC 3: Hover 상태
- Given 카드에 마우스 오버
- When 상태 확인
- Then border-indigo-300, bg-indigo-50, cursor-pointer

### AC 4: 선택 상태
- Given 사용자가 감정 카드 클릭
- When 클릭 실행
- Then 다음이 보임:
  - 카드: border-indigo-500, bg-indigo-50
  - 우측 상단: 파란 체크마크 (✓)
  - ring-4 ring-indigo-100

### AC 5: 선택 해제
- Given 이미 선택된 감정 카드를 다시 클릭
- When 클릭 실행
- Then 선택 상태가 해제됨 (원래 스타일로 돌아옴)

### AC 6: 다중 선택 불가 (단일 선택)
- Given 이미 "불안"을 선택한 상태
- When "자책" 카드 클릭
- Then "불안"은 자동 해제, "자책"이 새로 선택됨

### AC 7: 우측 요약 패널
- Given 감정 선택 후
- When 우측 패널 확인
- Then 다음이 보임:
  - "선택한 감정" 라벨
  - 선택된 감정의 이모지와 이름
  - 한 줄 설명 (선택적)

### AC 8: 상태 저장
- Given "불안"을 선택 후 이전 단계로
- When 2단계로 다시 돌아옴
- Then "불안"이 여전히 선택된 상태 (Zustand)
```

#### Issue RECORD-004: [3단계] 감정 강도 슬라이더
**복잡도**: 🟡 Mid | **소요시간**: 2.5시간

```markdown
# Issue: 3단계 - 감정 강도 슬라이더

## 목적
0-100 슬라이더로 감정의 강도를 입력

## 설계
- 큰 숫자 표시 (text-5xl, font-bold, 중앙)
- 슬라이더 바 (h-2, w-full)
- 3개 레이블: 0(낮음), 50(중간), 100(높음)
- 실시간 숫자 업데이트

## 완료 조건 (Acceptance Criteria)

### AC 1: 큰 숫자 표시
- Given 3단계 로드
- When 슬라이더 영역 확인
- Then 큰 숫자 (예: "80")이 중앙 상단에 text-5xl, font-bold로 표시

### AC 2: 슬라이더 바
- Given 숫자 아래
- When 슬라이더 확인
- Then 다음이 보임:
  - 배경 바: h-2, bg-gray-200, rounded-full
  - 채워진 부분: bg-indigo-600, 너비 = 값에 따라 변함
  - 썸: 원형, h-6 w-6, bg-indigo-600, 우측 border-4 border-white, shadow-md

### AC 3: 드래그 기능
- Given 슬라이더 썸을 드래그
- When 마우스로 이동
- Then 큰 숫자와 진행률이 실시간 변경

### AC 4: 클릭으로 값 설정
- Given 슬라이더 바를 클릭
- When 특정 위치 클릭
- Then 썸이 이동하고 값이 업데이트

### AC 5: 범위 제한
- Given 슬라이더 조작
- When 값 범위 확인
- Then 0 이상 100 이하로 제한됨

### AC 6: 레이블
- Given 슬라이더 바 아래
- When 텍스트 확인
- Then 3개 레이블 표시:
  - 좌측: "0  낮음"
  - 중앙: "50  중간"
  - 우측: "100  높음"
  - 색상: text-gray-500, text-sm

### AC 7: 저장 상태 배지
- Given 슬라이더 하단
- When "임시 저장됨" 배지 확인
- Then 초록 체크 아이콘 + 텍스트 "임시 저장됨" 표시

### AC 8: 상태 유지
- Given 슬라이더로 80 설정 후 이전 단계로
- When 3단계로 다시 돌아옴
- Then 슬라이더가 80 위치로 유지
```

#### Issue RECORD-005: [4단계] 자동 생각 입력
**복잡도**: 🟢 Low | **소요시간**: 1.5시간

```markdown
# Issue: 4단계 - 자동 생각 입력 (Textarea)

## 목적
그때 자동으로 든 생각을 텍스트로 입력

## 설계
- Textarea (min-h-[220px], rounded-2xl)
- 최대 500자
- 캐릭터 카운터

## 완료 조건 (Acceptance Criteria)

### AC 1: Textarea 렌더링
- Given 4단계 로드
- When 입력 필드 확인
- Then 큰 Textarea 표시:
  - placeholder: "그때 자동으로 든 생각을 적어주세요..."
  - min-h-[220px], rounded-2xl, border-gray-200

### AC 2: 입력 제한 (500자)
- Given Textarea에 포커스
- When 500자 이상 입력 시도
- Then 입력이 멈추고 경고 메시지 표시

### AC 3: 캐릭터 카운터
- Given Textarea에 입력
- When 텍스트 업데이트
- Then 우측 하단에 "150 / 500" 형태로 표시

### AC 4: 경고 색상 (400자 이상)
- Given 400자 이상 입력
- When 카운터 확인
- Then 색상이 주황색(warning)으로 변경

### AC 5: 다음 버튼 활성화
- Given 1글자 이상 입력
- When 버튼 확인
- Then 다음 버튼 활성화
```

### AC 5: 다음 버튼 활성화
- Given 둘 다 1글자 이상 입력
- When 버튼 확인
- Then 다음 버튼 활성화
