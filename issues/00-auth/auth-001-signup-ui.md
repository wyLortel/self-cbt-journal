## 완료 조건 (Acceptance Criteria)

### AC 1: 폼 렌더링
- **Given** 사용자가 /signup 접근
- **When** 페이지 로드
- **Then** 이메일, 비밀번호, 비밀번호 확인 입력창이 보임

### AC 2: 이메일 유효성 검사
- **Given** 이메일 입력창에 포커스
- **When** 유효하지 않은 이메일 입력 (예: "abc@")
- **Then** "유효한 이메일을 입력하세요" 에러 메시지 표시

### AC 3: 비밀번호 일치 검사
- **Given** 비밀번호 및 확인 입력창이 있는 상태
- **When** 두 비밀번호가 다르게 입력됨
- **Then** "비밀번호가 일치하지 않습니다" 에러 표시

### AC 4: 제출 버튼 로딩 상태
- **Given** 폼이 유효한 상태 (모든 필드 입력 완료)
- **When** 제출 버튼 클릭 → API 요청 중
- **Then** 버튼이 disabled되고 로딩 스피너 표시

### AC 5: 성공 시 리다이렉트
- **Given** 회원가입 API 호출 성공
- **When** 응답 수신 (user created)
- **Then** 자동으로 로그인 페이지 또는 대시보드로 리다이렉트

### AC 6: 에러 처리
- **Given** 회원가입 API 호출 실패 (예: 이미 존재하는 이메일)
- **When** 에러 응답 수신
- **Then** 에러 카드에 메시지 표시 (예: "이미 가입된 이메일입니다")

### AC 7: 모바일 반응형
- **Given** 모바일 기기에서 /signup 접근
- **When** 페이지 로드
- **Then** 버튼과 입력창이 전체 너비, 터치하기 쉬운 크기 (h-12)
