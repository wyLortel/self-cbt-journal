# spec-fixed.md

## 프로젝트명

셀프 마음 정리 기록 웹앱 (Self Mental)

---

## 기능 개요

불안, 자책, 후회, 분노 같은 힘든 감정이 들 때, 사용자가 **9단계 구조 (상황→감정→생각→사실/해석→인지왜곡→근거→대안→행동)** 에 따라 감정과 생각을 정리하고 저장할 수 있는 웹앱.

---

## 해결하려는 문제

**기존 방법의 한계:**

- 친구에게 말하면 위로는 받지만, 내 생각이 객관적인지 점검하기 어려움
- 명상이나 심리 영상은 도움이 되지만, 실제 감정이 올라오는 순간 적용이 어려움
- 일기장에 쓰면 구조 없이 자책 기록으로 끝나는 경우가 많음

**Self Mental이 해결하는 것:**

- 사실과 해석을 분리해서 생각하게 함
- 인지 왜곡을 구체적으로 인식하게 함
- 현실적인 대안 생각과 작은 행동으로 이어지게 함

---

## 대상 사용자

**Primary User:**

- 대학생/취업준비생
- 팀플, 발표, 피드백에서 쉽게 불안/자책에 빠지는 사람
- 상대 말투나 반응을 부정적으로 해석하는 경향이 있는 사람
- 상담은 부담스럽지만 혼자 마음을 정리하고 싶은 사람

**행동 특성:**

- 피드백 받으면 오래 곱씹음
- 메모나 명상만으로는 생각 정리가 안 됨
- "내 생각이 사실인가, 추측인가?" 누군가 정리해줬으면 좋다고 느낌

## 기능 요구사항

### 1. 사용자 인증

- 이메일 + 비밀번호 회원가입/로그인
- Google, GitHub 소셜 로그인 (선택적)
- 비밀번호 재설정 (이메일 기반)

### 2. 감정 기록 작성 (9단계)

- **1단계**: 상황 텍스트 입력 (무슨 일이 있었는가)
- **2단계**: 감정 선택 (불안, 자책, 수치심, 후회, 분노, 좌절, 외로움)
- **3단계**: 감정 강도 입력 (0~100 슬라이더)
- **4단계**: 자동 생각 텍스트 입력
- **5단계**: 사실/해석 분리 (사실은?, 내 해석은?)
- **6단계**: 인지 왜곡 7가지 중 다중 선택 (과잉일반화, 결론도약, 개인화, 확대/축소, 흑백사고, 라벨붙이기, 감정적추론)
- **7단계**: 찬성/반대 근거 입력
- **8단계**: 더 현실적인 대안 생각 입력
- **9단계**: 할 수 있는 작은 행동 하나 선택/입력

### 3. 기록 관리

- 저장된 기록 목록 조회 (최신순)
- 기록 수정 (9단계 모두 수정 가능)
- 기록 삭제 (재확인 후 삭제)
- 기록 재열기 (저장된 내용 다시 보기)

### 4. 기본 UI

- 로그인/회원가입 페이지
- 기록 목록 페이지 (감정 아이콘 + 날짜)
- 새 기록 작성 페이지 (9단계 폼)
- 마이페이지 (계정 정보, 로그아웃)

## 예외 상황 (Edge Cases)

### 1. 입력 값 검증

- 상황/생각 필드 빈 입력: "필수 항목입니다" 에러
- 감정 강도 범위 벗어남: 자동 조정 (0~100)
- 감정 선택 안 함: "최소 하나 선택해주세요" 에러

### 2. 네트워크 오류

- API 요청 실패: "다시 시도" 버튼 제공
- 타임아웃: 재시도 자동화
- 초안 자동 저장: 인터넷 재연결 후 계속 작성 가능

### 3. 동시 편집

- 같은 기록을 동시에 수정하려 하면: 최신 버전 우선 (CONFLICT 처리)

### 4. 세션 만료

- 30분 미활동 후 로그아웃
- 재로그인 후 작성 중인 내용 복구 (초안 저장된 경우)

### 5. 삭제 실수

- 기록 삭제 전 "정말 삭제하시겠습니까?" 확인 팝업
- 삭제된 기록은 복구 불가 (백업으로만 복구 가능)

## 기술 스택 (왜 이걸로 결정했는가?)

### 프론트엔드: React + TypeScript + Vite

**왜?**

- 대학생 타겟이므로 개발 속도 중요 → React의 풍부한 커뮤니티/라이브러리
- TypeScript로 감정 데이터 타입 안전성 확보
- Vite로 빠른 개발 환경

**대안 검토:**

- Vue: 더 배우기 쉽지만, 커뮤니티 작음
- Angular: 과도함

### 백엔드: Hono + TypeScript

**왜?**

- Node.js 경량 프레임워크 (Express는 구식)
- TypeScript 네이티브로 프론트엔드와 타입 공유 가능
- 현대적이고 빠름
- Cloudflare Workers/Vercel Edge도 지원

**대안 검토:**

- Express: 무겁고 구식
- Python FastAPI: 학습곡선 높음
- Firebase BaaS: 커스터마이징 제한

### DB: Supabase (PostgreSQL)

**왜?**

- Firebase 대체 (오픈소스 기반)
- 인증(Auth), RLS 통합 → 개발 시간 단축
- PostgreSQL 기반으로 복잡한 쿼리 가능
- 가격 저렴 (초기 무료)
- 실시간 구독 가능 (향후 확장)

**대안 검토:**

- Firebase: 벤더 락인, 커스터마이징 어려움
- MongoDB: 구조화된 데이터에는 부적합
- 자체 PostgreSQL: 관리 부담

### 배포: Vercel 또는 Railway

**왜?**

- 자동 배포 (Git push 시)
- Zero-config 호스팅
- 가격 저렴 (무료 tier 충분)
- 성능 좋음 (CDN 포함)

**대안 검토:**

- Cloudflare Workers: Hono 최적화이지만, 초기 학습곡선
- AWS: 복잡함, 비용 관리 어려움
- 자체 서버: 관리 부담

## 결정 이유 (Decision Rationale)

### 왜 9단계 구조인가?

- CBT(인지행동치료) 기반의 검증된 심리 치료 방법
- 단순 일기보다 구조화되어 있음
- 사용자가 "감정에 휩쓸리는 것"에서 "객관적 사고"로 전환

### 왜 인지 왜곡 7개인가?

- 10개 모두는 대학생 사용자에게 과부하
- 실제 대학생이 경험하는 상황에서 자주 나타나는 것 선별:
  - 과잉 일반화 (실패 경험 → 앞으로도 못함)
  - 결론 도약 (피드백 → 날 무시하는 거)
  - 개인화 (상대의 반응 → 내 때문)
  - 확대/축소 (작은 실수 → 끔찍한 일)
  - 흑백 사고 (완벽하지 않으면 → 쓸모없음)
  - 라벨 붙이기 (한 번의 실수 → 나는 루저)
  - 감정적 추론 (불안함 → 위험함)
- v1.1에서 사용자 피드백 후 추가 가능

### 왜 MVP는 1-9단계인가?

- 10단계 (감정 변화 추적)는 **기록이 충분히 쌓여야** 의미 있음
- MVP 빠른 출시가 우선 (사용자 피드백 수집)
- 10단계는 v1.1 로드맵으로 연기

### 왜 검색은 빼나?

- 초기 사용자는 기록이 적으므로 필요성 낮음
- 구현 복잡도 높음 (전문 검색 인덱싱 필요)
- 사용자가 요청할 때 추가하는 게 효율적

### 왜 Supabase + Vercel인가?

- 총 비용: $0 (초기 6개월)
- 개발 속도: 백엔드 개발 최소화
- 확장성: 사용자 증가 시 자동 스케일링
- Cloudflare Workers도 가능하지만, 개발 난이도 고려해 Vercel 추천

## 2. 기능 요구사항 (FRs)

### 2.1 사용자 인증

- **회원가입**: 이메일 + 비밀번호
- **소셜 로그인**: Google, GitHub (Supabase Auth 이용)
- **비밀번호 재설정**: 이메일 기반
- **로그아웃**: 세션 종료

### 2.2 감정 기록 작성 (1-9단계)

사용자가 다음 9단계 순서대로 감정을 정리하고 저장할 수 있어야 합니다.

#### **1단계: 상황 기록**

- 텍스트 입력 (최대 1000자)
- "지금 무슨 일이 있었는가?"

#### **2단계: 감정 선택**

- 선택지: 불안, 자책, 수치심, 후회, 분노, 좌절, 외로움 (7개)
- 단일 선택 (한 개만 선택 가능)

#### **3단계: 감정 강도**

- 슬라이더 또는 숫자 입력 (0~100)
- 시각적 피드백 (낮음/중간/높음)

#### **4단계: 자동 생각 기록**

- 텍스트 입력 (최대 500자)
- "그때 자동으로 든 생각은?"

#### **5단계: 사실 vs 해석 분리**

- **사실** 텍스트 입력: "실제로 일어난 객관적 사실"
- **해석** 텍스트 입력: "내가 추가로 해석한 부분"

#### **6단계: 인지 왜곡 체크**

- 다음 7가지 중 해당하는 것 **다중 선택**:
  1. **과잉 일반화** - "한 번 실패했으니 앞으로도 못할 거야"
  2. **결론 도약** - "근거 없이 나쁜 결과를 예상한다"
  3. **개인화** - "그건 내 때문이야"
  4. **확대/축소** - "작은 실수를 끔찍한 일로 생각한다"
  5. **흑백 사고** - "완벽하지 않으면 쓸모없다"
  6. **라벨 붙이기** - "나는 루저다 (스스로에게 부정적 낙인)"
  7. **감정적 추론** - "불안하니까 실제로 위험한 거겠지"

#### **7단계: 근거와 반대 근거**

- **찬성 근거** 텍스트: "내 생각을 뒷받침하는 근거"
- **반대 근거** 텍스트: "내 생각에 반대하는 근거나 다른 해석"

#### **8단계: 대안 생각**

- 텍스트 입력 (최대 500자)
- "더 현실적이고 균형잡힌 생각은?"

#### **9단계: 작은 행동**

- 텍스트 입력 (최대 200자)
- "지금 할 수 있는 작은 행동 하나는?"

### 2.3 기록 관리

- **저장**: 각 단계를 완료할 때마다 임시 저장 (草稿 형태)
- **제출**: 9단계 완료 후 최종 저장
- **수정**: 저장된 기록을 다시 열어서 수정 가능
- **삭제**: 기록 삭제 가능 (재확인 필요)
- **조회**: 본인의 모든 저장된 기록 목록 조회 (최신순)

### 2.4 기록 목록

- 저장된 기록 시간순 표시
- 감정 아이콘 + 감정명 + 날짜/시간 표시
- 한 줄 요약 표시 (상황 또는 감정 강도)

## 5. 데이터 모델

### 5.1 테이블 구조

#### **users** (Supabase Auth 기본 테이블)

```sql
id: UUID (primary key)
email: string (unique)
created_at: timestamp
updated_at: timestamp
```

#### **emotion_records** (감정 기록)

```sql
id: UUID (primary key)
user_id: UUID (foreign key → users.id)
created_at: timestamp
updated_at: timestamp
deleted_at: timestamp (논리 삭제)

-- 1단계: 상황
situation: text

-- 2단계: 감정
emotion: enum('불안', '자책', '수치심', '후회', '분노', '좌절', '외로움')

-- 3단계: 감정 강도
intensity: integer (0-100)

-- 4단계: 자동 생각
automatic_thought: text

-- 5단계: 사실 vs 해석
fact: text
interpretation: text

-- 6단계: 인지 왜곡 (배열)
cognitive_distortions: text[]
  -- e.g., ['과잉일반화', '결론도약']

-- 7단계: 근거
supporting_evidence: text
contradicting_evidence: text

-- 8단계: 대안 생각
alternative_thought: text

-- 9단계: 작은 행동
small_action: text

-- 상태
status: enum('draft', 'completed')
```

### 5.2 인덱스

- `emotion_records(user_id, created_at)` - 사용자별 기록 조회 성능
- `emotion_records(user_id, deleted_at)` - 삭제된 기록 필터링

## 6. API 명세 (개요)

### 인증 API (Supabase Auth 기본 제공)

- `POST /auth/signup` - 회원가입
- `POST /auth/login` - 로그인
- `POST /auth/logout` - 로그아웃
- `GET /auth/user` - 현재 사용자 정보

### 감정 기록 API

#### 기록 작성/수정

```
POST /api/records - 새 기록 생성
PATCH /api/records/{id} - 기록 수정
GET /api/records/{id} - 기록 조회
DELETE /api/records/{id} - 기록 삭제

GET /api/records - 사용자의 모든 기록 (페이징)
```

### API 응답 형식

```json
{
  "success": true,
  "data": { ... },
```

## 8. MVP vs v1.1 로드맵

### MVP (현재)

✅ 사용자 인증
✅ 1-9단계 기록 작성/수정/삭제
✅ 인지 왜곡 7가지 체크
✅ 기록 목록 조회

### v1.1 (이후)

- ⏳ 10단계: "다시 돌아보며 감정 변화 확인"
  - 시간대별 기록 비교
  - 감정 강도 변화 차트
- ⏳ 기록 검색/필터링 (감정별, 날짜별)
- ⏳ 통계 대시보드 (가장 많은 감정, 트렌드)
- ⏳ 사용자 데이터 다운로드/삭제
- ⏳ 초대 기능 또는 공개 설정 (선택)

### v2.0+ (장기)

- 심리 상담사 연계 기능
- 감정 추천 알고리즘 (AI)
- 커뮤니티 공유 (익명)

## 9. 성공 지표 (Success Metrics)

### MVP 성공 기준

- [ ] 회원가입 사용자 10명 이상
- [ ] 주당 3회 이상 기록 작성하는 사용자 비율 > 50%
- [ ] 기록 완성률 (9단계 모두 작성) > 70%
- [ ] 사용자 만족도 (NPS) > 30

### 피드백 수집

- 인지 왜곡 7가지 중 자주 선택되는 항목
- 사용자가 추가를 원하는 왜곡
- 사용성 이슈 (폼 완성도, UI/UX)

## 10. 제약 사항 (Constraints)

- **데이터 개인성**: 감정 기록은 극도로 개인적 → 보안 최우선
- **정신 건강 앱 아님**: 전문 상담 대체가 아님 → 면책 조항 필수
- **심리학 정확성**: 인지 왜곡 정의는 CBT 기반이나, 모든 상황을 다루지 않음 → v1.1에서 확장
